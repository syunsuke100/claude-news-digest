# Claude News Digest

Anthropic のAI「Claude」に関するニュースを毎朝自動収集し、AIによる日本語要約と
図解付きのダイジェストを GitHub Pages で公開するツールです。

- **完全無料で運用できます**(GitHub Actions + GitHub Pages + 各社LLMの無料枠)
- **サーバー不要**。すべて GitHub 上で完結します
- 公開されるページの URL をスマートフォンのホーム画面に登録しておけば、
  毎朝ワンタップでその日のダイジェストを読めます

> ⚠️ 本プロジェクトは個人が開発した非公式ツールであり、Anthropic 社とは関係ありません。

## できること

毎朝 7:00(JST)に、直近48時間の Claude 関連ニュース(日本語・英語)を集めて、
1枚のページにまとめます。ページの構成は上から次のとおりです。

1. **まとめ** — その日の動向を横断した日本語の見出しと3〜4文の要約
2. **図解① トピック分類** — 記事を5カテゴリ(新モデル・アップデート / Claude Code・開発ツール / 規制・ポリシー・安全性 / 研究・技術 / その他)に分類した件数グラフ
3. **図解② 重要トピック** — 特に重要な話題2〜4個の一覧
4. **参照記事リスト** — カテゴリタグ付きの記事一覧(最大20件、元記事へのリンク)

英語記事は日本語に翻訳されたうえで要約・分類に含まれます。
ヘッダーには「取得 X 件 → 掲載 Y 件」と表示され、収集した記事のうち
何件が無関係・重複として除外されたかが分かります。

過去のダイジェストは `docs/archive/` に日付ごとに保存されます。

## 仕組み

```
GitHub Actions(毎朝 07:00 JST に自動実行)
  └─ main.py
       ├─ Google ニュース RSS(日本語+英語)から直近48時間の記事を取得
       ├─ LLM に一括投入し、JSON で受け取る:
       │    無関係記事の除外・重複統合・翻訳・全体要約・トピック分類・重要トピック抽出
       ├─ 受け取った JSON をもとに、コードが HTML と SVG 図解を生成
       ├─ docs/index.html を更新(GitHub Pages が公開)
       ├─ docs/archive/YYYY-MM-DD.html に過去ログを保存
       └─ 変更を自動コミット
```

設計上のポイント:

- **HTML・図解は LLM に書かせません。** LLM の役割は「翻訳・要約・分類」を
  構造化データ(JSON)で返すところまでで、ページの描画はすべてコード側で行います。
  レイアウトが日によって崩れず、トークン消費も抑えられます。
- **LLM プロバイダは差し替え可能です。** gemini / groq / mistral に対応しており、
  どのプロバイダ・モデルを使うかは GitHub の Secrets(環境変数)だけで決まります。
  コードに既定値は書かれていないため、リポジトリを見ても実際に使われている
  プロバイダ・モデルは分かりません。プロバイダの追加も関数を1つ足すだけです。
- **LLM が使えない日も壊れません。** 障害や無料枠の上限超過などで要約に失敗した
  場合は、まとめ・図解なしの記事一覧のみのページに自動で切り替わります。

## 自分用に動かす

このリポジトリを自分のアカウントに複製(フォークまたはテンプレートとして利用)
すれば、同じ仕組みを自分の GitHub Pages で運用できます。

### 1. LLM の API キーを用意する

gemini / groq / mistral のいずれかで API キーを取得してください。
いずれも無料枠の範囲で運用できる想定です(1日1回・1リクエストの軽い用途です)。
無料枠の条件や対象モデルは各社とも変わることがあるため、
取得時に最新の条件を確認してください。

### 2. Repository secrets を3つ登録する

リポジトリの **Settings → Secrets and variables → Actions →
New repository secret** から、次の3つを登録します。

| Name | 値 | 説明 |
|---|---|---|
| `LLM_PROVIDER` | `gemini` / `groq` / `mistral` | 使用するプロバイダ |
| `LLM_MODEL` | プロバイダごとのモデル名 | 使用するモデル |
| `LLM_API_KEY` | 取得した API キー | 認証キー |

- 3つとも必ず **Secrets** に登録してください(**Variables は平文表示されるため不可**)。
- Secrets はリポジトリが Public でも第三者から見えず、Actions のログでも
  自動でマスクされます。フォークからの Pull Request にも渡されません。
- 未設定のまま実行すると、設定エラーとして明示的に停止します。

### 3. GitHub Pages を有効化する

**Settings → Pages** で次のように設定します。

- Source: **Deploy from a branch**
- Branch: `main` / フォルダ `/docs`

数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でページが公開されます。

### 4. 初回実行

**Actions タブ → Claude News Digest → Run workflow** で手動実行します。
成功すると `docs/index.html` が生成され、以降は毎朝 07:00 JST に自動更新されます。

### 5. スマートフォンのホーム画面に登録(推奨)

- iPhone(Safari): 公開 URL を開く → 共有 → 「ホーム画面に追加」
- Android(Chrome): 公開 URL を開く → メニュー → 「ホーム画面に追加」

## カスタマイズ

| 変更したいこと | 場所 | 既定値 |
|---|---|---|
| 収集する検索キーワード | `NEWS_QUERY`(`daily.yml` の env) | `Anthropic Claude` |
| 収集対象の時間幅 | `main.py` の `LOOKBACK_HOURS` | 48 時間 |
| 掲載する最大記事数 | `main.py` の `MAX_ITEMS` | 20 件 |
| トピック分類のカテゴリ | `main.py` の `CATEGORIES` | 5カテゴリ |
| 更新時刻 | `daily.yml` の cron(UTC 指定) | `0 22 * * *`(= 07:00 JST) |

- 検索キーワードを変えれば、Claude 以外のテーマのデイリーダイジェストとしても
  そのまま使えます。
- プロバイダ・モデルの変更は Secrets の値を差し替えるだけで、コードの変更は不要です。
- 対応プロバイダを増やしたい場合は、`main.py` に `_call_xxx` 関数を1つ追加して
  `_PROVIDERS` に登録してください。呼び出し側の変更は不要です。

## ローカルでの動作確認

```bash
pip install -r requirements.txt
export LLM_PROVIDER=xxxx
export LLM_MODEL=xxxx
export LLM_API_KEY=xxxx
python main.py          # docs/index.html が生成されます
```

## 制限事項・注意点

- **Google ニュースの RSS は非公式の仕組みです。** 長年安定して動作していますが、
  公式ドキュメントはなく、予告なく仕様が変わる可能性があります。
- **記事リンクは Google ニュースのリダイレクト URL のままです。** タップすれば
  元記事に転送されるため閲覧には支障ありませんが、URL 自体は元記事のものでは
  ありません。
- **各社 LLM の無料枠は変更されることがあります。** 上限超過時は記事一覧のみの
  表示に自動で切り替わりますが、恒常的に超える場合はプロバイダやモデルの
  見直しが必要です。
- **要約は AI によるものです。** 内容の正確性は保証されません。重要な情報は
  必ずリンク先の元記事で確認してください。
- GitHub の scheduled workflow は混雑状況により実行が数分〜十数分遅れることが
  あります。
- 収集したニュースの利用は、私的な閲覧の範囲にとどめることを想定しています。
  各記事の著作権は各媒体に帰属します。
